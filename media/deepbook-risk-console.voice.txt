DeepBook Risk Console. A wallet-connected DeepBookV3 risk console that turns price predictions and LP guardrails into tiny guarded testnet maker orders with on-chain receipts. The old workflow is broken. Sui has a powerful native order book, but builders need safer tools to translate strategies into auditable execution plans. DeepBook Risk Console packages that into a focused MVP: pool risk console and predict scenario scorer. DeepBook-native UX can turn liquidity infrastructure into judge-visible products with real differentiation. In this demo the app opens locally from the submission repo. It starts on a dashboard, saves a policy, checks an agent request, creates an embedded Sui wallet, requests local SUI, and signs the transaction. That is the important distinction. The user does not need to install a browser wallet for the recording, and the judge does not have to trust a fake success screen. The visible workflow is deliberately step by step: dashboard, policy, agent request, wallet approval, and receipt. The product boundary is real: an intent becomes policy state, wallet state, execution state, and an inspectable receipt. One object, one Move call, one digest per run. The localnet package owns state, the embedded wallet signs, and the digest becomes the receipt. That matters because Sui gives the project explicit objects, fast finality, readable transaction effects, and a receipt path judges can inspect. The same pattern can be extended after the hackathon into spending limits, approvals, refunds, storage attestations, or strategy execution depending on the track. The folder ships with the app, Move package, SDK adapter, localnet script, generated subtitles, visual assets, and this pitch video. Judge it as a shippable DeepBook wedge, not a slide deck.
